# Linux进程通信：实现自己的shell

## 什么是shell？

 `shell`实际上就是一个命令行解释器，它能够接收用户输入的命令，解析并执行这些命令。

* **shell的执行流程**：

* * **输入命令**：在终端输入命令，shell读取并解析。<br>
* * **创建进程**：shell使用`fork()`创建一个子进程，子进程负责执行用户输入的命令。<br>
* * **执行命令**：子进程通过`exec()`函数加载并执行命令。<br>
* * **输出结果**：子进程将结果输出到终端，shell输出提示符等待下一条命令。

## 主要使用的系统调用：

#### fork()： 创建子进程，子进程获得父进程的栈、数据段、堆和执行文本段的拷贝。

* 子进程的内容一开始是对父进程个部分的完全复制，执行后可各自修改各自的数据，不相互影响。<br>
* 通过`fork`的返回值区分子、父进程，以及是否创建成功。<br>
* * 父进程调用后返回新创建的子进程的进程ID。
* * 子进程调用后返回0.
* * 创建进程失败，返回-1。

#### 库函数exit(status)：终止进程，将进程占有的所有资源，包括内存、文件描述符等怪还内核，再次分配。`status`是一个整形变量，表示进程的退出状态

#### exec()库函数：用于替换当前进程的映像为新程序，shell会在子进程中使用exec()来加载并执行用户输入的命令。

#### pipe():创建一个管道，用于连接多个进程的输入输出，创建两个文件描述符，分别用来读和写。

用来在父进程和子进程之间传递数据，通常和`fork()`、`exec()`连用，用于实现管道功能。

#### mkfifo()：用于创建命名管道，是一个特殊的文件类型，用于在两个进程间传递数据。


## shell的基本框架

 `shell`本质上是一个循环：
 打印提示符 --> 读取用户输入 --> 解析输入命令和参数 --> 创建子进程执行命令 --> 等待命令执行完成 --> 返回提示符，继续下一轮循环

## 使用的头文件和函数

* **头文件**：
```c
    #include <stdio.h>
    #include <stdlib.h>
    #include <string.h>
    #include <unistd.h>   // POSIX系统调用，如 fork() execvp() getcwd()
    #include <sys/wait.h> // 等待子进程结束，如 wait() waitpid()
```

* **函数**：
```c
fgets()       // 从标准输入读入一行命令

strtok()      // 将输入命令按空格拆分为命令和参数

execvp()      // 执行外部命令，替换当前子进程映像

```

## 输入输出重定向

* 输出重定向：

    `>`：覆盖写入文件 如果文件存在，则清空原文件；如果不存在，则新建文件

    ```c
    echo "hello" > file.txt

    int fd = open("file.txt", O_WRONL | O_CREAT | O_TRUNC, 0644); // O_TRUNC 为覆盖
    dup2(fd, STDOUT_FILENO); // 重定向到 stdout 文件
    close(fd);
    ```

    `>>`：追加写入文件 保留原本内容，将新内容加到末尾 

    ```c
    echo "world" >> file.txt

    int fd = open("file,txt", O_WRONL | O_CREAT | O_APPEND, 0644); // O_APPEND 为追加
    close(fd);
    ```

* 输入重定向：

    `<`：将文件内容作为命令的标准输入。

    ```c
    wc -l < file.txt

    int fd = open("file.txt", O_RDONLY);
    dup2(fd, STDIN_FILENO); // 重定向 stdin 到文件
    close(fd);
    ```

### 实现在 shell 中实行：cat < input.txt > output.txt

    input.txt 文件作为 cat 的输入； cat 输出 重定向到 output.txt； 终端不会显示任何内容

    ```c
    int fd_in = ("input.txt", O_RDONLY);
    int fd_out = ("output.txt", O_WRONL | O_CREAT | O_TRUNC);

    dup2(fd_in, STDIN_FILENO);
    dup2(fd_out, STDOUT_FILENO);

    char *argv[] = {"cat", NULL};
    execvp("cat", argv);
    ```

## 何为管道 |

    管道将一个命令的输出作为另一个命令的输入。

    创建管道得到两个文件描述符，分别为读端和写端； 
    假设有两个子进程，子进程1和子进程2；
    在子进程1中将输出重定向到写端，在2中将写端重定向到读端；
    关闭管道中不需要的端口，执行两个命令。

#### 多管道

    如果有三个命令：

    ` ls -l | grep txt | wc -l `

    需要两个管道 “命令1 | 命令2 | 命令3”

    数据流动 “ ls -l 输出 --> grep txt 输入/输出 --> pipe2 --> wc -l 输入 --> 终端”

    解决办法：
    1. 为每对命令创建一个管道；
    2. 使用 dup2 重定向 stdout/stdin
    3. fork 每个子进程
    4. 父进程等待所有子进程

## 后台执行 &

    在 bash 中可以执行：“sleep 10 &”

    此命令在后台运行，终端立即返回提示符，后台任务的标准输出仍然可以输出到终端

    c语言执行思路：

    检查命令末尾是否有 &； 如果有，在父进程不等待子进程； 如果没有，父进程调用 waitpid 等待子进程结束

    ```c
    int back = 0; 
    int lin = strlen(cmd);
    if(cmd[len-1] == '&') {
        back = 1; // back = 1 表示后台运行
        cmd[len-1] = '\0'; // 去掉 &
    } 

    pid_t pid = fork();
    if(pid == 0) {
        execvp(argv[0], argv);
        perror("exec failed");
        exit(1);
    } else if(pid > 0) {
        if(!back) {
            waitpid(pid, NULL, 0) // 前台等待 父进程等待子进程
        } else {
            printf("[后台] pid=%d\n", pid);
        }
    } else {
        perror("fork failed");
    }
    ```

## 内置命令 cd 加强

    cd <目录> --> 切换到制定目录
    cd ~ --> 回到 home
    cd - --> 回到上一次目录
    支持 相对路径 和 绝对路径

```c
char prev_dir[1024] = "";
char current_dir[1024];

getcwd(current_dir, sizeof(current_dir)); // getcwd 获取当前目录

if(strcmp(argv[0], "cd") == 0) {
    char *target = argv[1];
    if (!target || strcmp(target, "~") == 0) {
        target = getenv("HOME");
    } else if (strcmp(target, "-") == 0) {
        target = prev_dir; // prev_dir 保存上一次目录
    }

    if (chdir(target) != 0) { // chdir 切换目录
        perror("cd failed");
    } else {
        strcpy(prev_dir, current_dir);
    }
    continue;
}
```

## 信号处理

* **目的**：

* * 防止用户按 ctrl+c 终止 shell
* * 让 shell 忽略一些信号，而子进程仍然可以被 ctrl+c 终止

```c
#include <signal.h>

signal(SIGINT, SIG_IGN); // 忽略 ctrl+c
```
使 shell 自身不被终止
子进程默认可以接收 ctrl+c 

```c
if (pid == 0) {
    signal(SIGINT, SIG_DFL); // 子进程使用默认信号
    execvp(argv[0], argv);
}
```

## 美化 shell 提示符

#### 基本提示符 

` printf("my-shell:%s$ ",cmd); `

显示当前工作目录 cwd <br>
显示一个固定前缀 my-shell

#### 加入用户名和主机名

```c

char *username = getpwuid(getuid())->pw_name; //  获取当前用户
char hostname[64];
gethostname(hostname, sizeof(hostname)); // 获取主机名

printf("%s@%s:%s$ ", username, hostname, cwd);
```

` printf("\033[1;32m%s@%s\033[0m:\033[1;34m%s\033[0m$ ", username, hostname, cwd); // 用户名绿色，目录蓝色 `

## 错误处理

1. 命令不存在

```c
execvp(argv[0], argv);
perror("执行命令失败");
```

2. 目录不存在

```c
if (chdir(target) != 0) {
    perror("cd failed");
}
```

3. 打开文件失败

```c
int fd = open("output.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
if (fd < 0) {
    perror("open failed");
    continue;
}
```

* **错误处理原则**：

* * 尽量让 shell 继续运行，而不是直接退出

* * 给用户提示信息，但不要崩溃
